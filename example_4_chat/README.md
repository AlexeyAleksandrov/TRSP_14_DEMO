# Пример 4: Чат с событийной моделью

## Описание

Данный пример демонстрирует реализацию чата с использованием паттерна ConnectionManager и событийной модели, близкой к Qt.

## Событийная модель

### Обработчики событий

- `on_connect` - вызывается при подключении клиента
- `on_receive` - вызывается при получении сообщения
- `on_disconnect` - вызывается при отключении клиента

### Пример регистрации обработчика

```python
@manager.on_connect
async def handle_connect(websocket, room, username):
    print(f"{username} подключился к {room}")
```

## Возможности

- Подключение к комнатам
- Отправка сообщений всем участникам
- Отправка личных сообщений
- Уведомления о входе/выходе участников
- Список участников комнаты

## Структура

- `server.py` - сервер с ConnectionManager
- `client.py` - интерактивный клиент / демо-режим

## Запуск

### Сервер

```bash
uvicorn server:app --reload --port 8000
```

### Демо-режим (автоматическая демонстрация)

```bash
python client.py
```

### Интерактивный режим

Терминал 1:
```bash
python client.py general Алиса
```

Терминал 2:
```bash
python client.py general Борис
```

## Формат сообщений

### Общее сообщение

```json
{"тип": "общее", "сообщение": "Привет всем!"}
```

### Личное сообщение

```json
{"тип": "личное", "кому": "Алиса", "сообщение": "Привет!"}
```
