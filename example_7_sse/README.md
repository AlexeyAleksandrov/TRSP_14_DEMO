# Пример 7: Server-Sent Events (SSE) для уведомлений

## Описание

Данный пример демонстрирует использование SSE для отправки уведомлений о новых сообщениях. Один клиент отправляет HTTP-запрос с сообщением, второй получает уведомление через SSE.

## Сценарий использования

```
┌────────────┐                    ┌──────────┐                    ┌────────────┐
│  Клиент A  │                    │  Сервер  │                    │  Клиент B  │
│ (Отправ.)  │                    │          │                    │ (Получ.)   │
└─────┬──────┘                    └────┬─────┘                    └─────┬──────┘
      │                                │                                │
      │                                │◄──── SSE подключение ──────────│
      │                                │                                │
      │── POST /send ─────────────────►│                                │
      │   (sender, receiver, content)  │                                │
      │                                │                                │
      │◄── {статус: доставлено} ───────│                                │
      │                                │                                │
      │                                │── event: message ─────────────►│
      │                                │   data: {от, кому, сообщение}  │
      │                                │                                │
```

## Структура

- `server.py` - сервер с SSE и HTTP-эндпоинтами
- `client_sse.py` - SSE-клиент для получения уведомлений
- `client_sender.py` - HTTP-клиент для отправки сообщений

## Эндпоинты

- `GET /sse/{client_id}` - SSE-поток для получения уведомлений
- `POST /send?sender=...&receiver=...&content=...` - отправка сообщения
- `POST /broadcast?sender=...&content=...` - рассылка всем
- `GET /clients` - список подключённых клиентов

## Запуск

### Терминал 1: Сервер

```bash
uvicorn server:app --reload --port 8000
```

### Терминал 2: SSE-клиент (получатель)

```bash
python client_sse.py Алиса
```

### Терминал 3: Отправка сообщения

```bash
python client_sender.py send Борис Алиса "Привет, Алиса!"
```

### Рассылка всем

```bash
python client_sender.py broadcast Система "Важное объявление!"
```

## Когда использовать SSE вместо WebSocket

SSE подходит для:

- Уведомлений от сервера
- Лент новостей
- Биржевых котировок
- Обновлений статусов

WebSocket предпочтительнее когда:

- Нужна двусторонняя связь в реальном времени
- Клиент активно отправляет данные
- Требуется передача бинарных данных
