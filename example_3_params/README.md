# Пример 3: Маршруты WebSocket с параметрами

## Описание

Данный пример демонстрирует различные способы передачи параметров в WebSocket-соединение.

## Способы передачи параметров

### Path-параметры

Передаются непосредственно в URL:

```
ws://localhost:8000/ws/room123
```

### Query-параметры

Передаются после знака `?`:

```
ws://localhost:8000/ws/chat/room1?username=Ivan&token=secret
```

### Через первое сообщение

WebSocket не поддерживает body при рукопожатии. Данные, которые обычно передавались бы в теле HTTP-запроса, можно отправить первым сообщением после установки соединения.

## Структура

- `server.py` - сервер с тремя эндпоинтами для демонстрации параметров
- `client.py` - клиент для тестирования всех способов

## Эндпоинты

- `/ws/{room_id}` - только path-параметр
- `/ws/chat/{room_id}?username=...&token=...` - path + query параметры
- `/ws/init` - инициализация через первое JSON-сообщение

## Запуск

### Сервер

```bash
uvicorn server:app --reload --port 8000
```

### Клиент

```bash
python client.py
```

## Зависимости

```
fastapi
uvicorn
websockets
```
