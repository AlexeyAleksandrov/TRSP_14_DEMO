# Пример 5: Передача файлов по WebSocket

## Описание

Данный пример демонстрирует передачу файлов через WebSocket, включая чанкованную передачу больших файлов.

## Принцип работы

При передаче больших файлов данные разбиваются на чанки (по умолчанию 64 КБ). Это аналогично работе QTcpSocket в Qt, который также разделяет большие данные на несколько пакетов.

### Протокол передачи

1. Отправитель сообщает метаданные файла (имя, размер, количество чанков)
2. Получатель подтверждает готовность к приёму
3. Отправитель последовательно отправляет чанки
4. Получатель подтверждает каждый чанк
5. После получения всех чанков файл собирается и проверяется контрольная сумма

### Формат сообщений

Начало передачи:
```json
{
  "тип": "начало_передачи",
  "id": "uuid",
  "имя_файла": "file.txt",
  "размер": 204800,
  "количество_чанков": 4
}
```

Чанк (метаданные + бинарные данные):
```json
{"тип": "чанк", "id": "uuid", "индекс": 0}
```
За которым следует бинарное сообщение с данными.

## Структура

- `server.py` - сервер с эндпоинтами для загрузки и скачивания
- `client.py` - клиент с функциями upload/download

## Эндпоинты

- `/ws/upload` - загрузка файлов на сервер
- `/ws/download/{filename}` - скачивание файлов с сервера

## Запуск

### Сервер

```bash
uvicorn server:app --reload --port 8000
```

### Демонстрация

```bash
python client.py
```

### Демонстрация большого файла (чанкование)

```bash
python client.py large
```

### Загрузка файла

```bash
python client.py upload myfile.txt
```

### Скачивание файла

```bash
python client.py download myfile.txt local_copy.txt
```
